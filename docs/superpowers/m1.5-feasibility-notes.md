# M1.5 feasibility gate — notes (2026-05-31)

The §8 feasibility gate from `docs/superpowers/specs/2026-05-31-m1.5-raster-asset-design.md`, run on the owner's RTX 3070 8 GB. Purpose (spec §8): stand the local stack up and surface infra/stack findings **before** the proof runs, so any gap is attributable to the stack, not the skill.

## ✅ GATE CLEARED (2026-05-31 — RTX 5080, torch 2.11.0+cu128)

**The "open blocker" (finding #5, gray interior) was a misdiagnosis. The LayerDiffuse decoder works perfectly.** Resolution:

- New card: **RTX 5080 16 GB (Blackwell, sm_120)**. Blackwell requires **torch ≥2.7 / cu128**; the existing venv (torch 2.6.0+cu124) topped out at sm_90 and could not run kernels. Upgraded the ComfyUI venv to **torch 2.11.0+cu128** (`pip install --upgrade torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128`). `arch_list` now includes `sm_120`; GPU matmul verified. **This also kills the old "version-bisect *down* to the node's 2024-era stack" escape hatch — Blackwell can't go below cu128 — but it turned out not to be needed.**
- Ran the node's own `layer_diffusion_fg_example_rgba.json` verbatim (API port, Juggernaut v9, `_gate_diag.py`), saving BOTH node 14 (plain VAEDecode) and node 36 (`LayeredDiffusionDecodeRGBA`). First run looked like the gray failure again — but inspecting the **alpha channel** showed it was **inverted** (subject transparent, background opaque). Every prior "gray interior" measurement had been reading the flat **background** as the subject. The decoder's RGB, measured over the *correct* (inverted) mask, had luma std **72.8 — identical to the plain render (71.9)**. Decoder never broken.
- **Root cause = a one-line sign error in our own join-patch.** `LayeredDiffusionDecodeRGBA.decode` did `alpha = 1.0 - mask`, but the parent `decode` already returns opacity (channel 0) as its second value. Corrected to use it directly (`image, alpha = super().decode(...)`, no inversion). Re-ran end-to-end: node 36 now emits a correct RGBA sprite (vivid interior std 73.05, clean transparent bg, 17.3% subject coverage). **VERDICT: PASS.**
- Throughput on the 5080 with no offload: a 1024² master generates in ~20–40 s (vs 8–12 min on the 3070).

## Findings (in the order they surfaced)
1. **Alpha-join API drift (fixed).** `LayeredDiffusionDecodeRGBA` called `JoinImageWithAlpha().join_image_with_alpha(...)`, but current ComfyUI made `JoinImageWithAlpha` a v3 `io.ComfyNode` (method renamed, and it re-inverts alpha internally). The whole pipeline ran and only the final join threw. **Fix:** patched the node's `decode` to build the RGBA tensor directly (`torch.cat([rgb, alpha], -1)`), preserving the original single-inversion behavior.
2. **LoRA-patch incompatibility → ComfyUI pinned to v0.3.16 (fixed).** A `--depth 1` clone pulled bleeding-edge ComfyUI (2026-05). Its `comfy/lora.py:calculate_weight` only recognizes `diff`/`set`/`model_as_lora` patch types (LoRAs now go through `weight_adapter` objects). LayerDiffuse emits raw `("lora", weight_list)` tuples → `patch type not recognized` warnings, so the **attention/conv-injection patch was silently dropped** and the transparent decoder produced haze. **Fix:** checked out **ComfyUI v0.3.16** (commit `26c7baf`, 2026-02-26 — contemporaneous with the node's last commit 2026-02-25), whose `calculate_weight` handles `lora`/`lokr`/`loha`/`glora` inline. Warnings gone; patch applies.
3. **"Mud" → Conv Injection + canonical settings (fixed).** Used `"SDXL, Attention Injection"` + cfg 6.5 + scheduler `karras`. The node's own FG-RGBA examples all use **`"SDXL, Conv Injection"`** + cfg 8 + scheduler `normal` + euler. Switching produced a correct, recognizable creature **silhouette**. (Templates updated to Conv Injection + `normal`.)
4. **Base SDXL 1.0 is flat (mitigated).** Base 1.0 gives soft, low-detail fills; the node examples use a finetune (**Juggernaut XL**). Downloaded Juggernaut XL v9. Plain (no-LayerDiffuse) Juggernaut renders are excellent (luma std ~62, a gorgeous painterly forest deer).
5. **OPEN — transparent decoder desaturates the interior.** Through `LayeredDiffusionDecodeRGBA`, the **alpha is perfect but the RGB interior collapses to flat gray** (luma std ~13–17 vs ~62 plain), color surviving only at the silhouette edge. The decoder is a trained `UNet1024(pixel, latent)` (8 flip/rot passes, median) fed the raw `samples["samples"]` latent + VAE pixel; output convention is `[0,1]` (per `fill_checkerboard_bg`). **Ruled out by experiment (each ~10 min on 8GB):** checkpoint (base 1.0 vs Juggernaut XL v9 — identical gray); decoder precision (forcing fp32 — no change); **diffusers version** (0.38.0 → pinned node-era 0.29.2 — no change); **latent scale** (÷0.13025 — *worse*, std 13). Reasoned out: pixel range (decoder I/O is `[0,1]`, and the plain render is vivid). Web evidence (huchenlei/ComfyUI-layerdiffuse issues) shows the node **does** work for others on SDXL — the reported failures are anime-finetune *transparency* failures, not gray interiors — so our environment differs in some dimension (candidates: **torch 2.6.0 / transformers 5.9 too new**, or ComfyUI still newer than the node's true functional target ~early-2024). This is the genuine open blocker for usable raster art; next attack = methodical version bisection on the 16GB card with a known-good reference to compare against, or a maintained LayerDiffuse fork / `sd-forge-layerdiffuse`.

## Recommended next step (decisive, cheap-ish)
Run the node's **own bundled example** `custom_nodes/ComfyUI-layerdiffuse/example_workflows/layer_diffusion_fg_example_rgba.json` verbatim (swap in `juggernautXL_v9`), converted to API format, and inspect the RGB interior:
- If it is **also gray** → the node is broken against this whole stack; try a different ComfyUI commit bracketing the node's release, a maintained LayerDiffuse fork, or the reference `sd-forge-layerdiffuse`.
- If it is **colored** → our `comfy.mjs` workflow differs in a detail (latent wiring/scale); diff against the example and fix the template.
Best done **after the 16 GB VRAM upgrade** — generation on 8 GB offloads heavily (~15–30 s/step, ~8–12 min per 1024² master), which also tripped `comfy.mjs`'s 600-poll (10-min) ceiling; bump `maxPolls` or rely on 16 GB speed.

## Throughput finding
8 GB forces constant model offload (load log: ~5.6 GB usable). 16 GB fits SDXL + LayerDiffuse at 1024² with no offload → expected ~10–30× faster (seconds, not minutes per sprite). This is the single biggest practical unblock for the proof runs.
